1DDH 启动机制

DDH 默认 Standby（仅 emergency 兜底）。CIO 主观或市场客观触发后进入 Active。

1.1 状态机

[Standby] ──┬── CIO toggle ON ──→ [Active · Standard] ├── 单一中级市场触发 ──→ [Active · Standard] ├── 多重中级 (≥2) 或 单高级触发 ──→ [Active · Defensive] └── 多重高级 (≥2) 触发 ──→ [Active · Emergency 手工] [Active] ──── CIO OFF + 所有市场触发解除 + 1h cool-down ────→ [Standby]

1.5 Standby 模式的 emergency 兜底

DDH OFF ≠ 不 hedge。以下条件强制升级到 Active：
  • Γ_p ≥ 0.30（Orange）→ 强制减 Γ（见 §2.2）
  • Δ_net 漂移 > ±10% NAV（约 ±2000 BTC @ 当前 NAV 20000 BTC）
  • 跳空 1h |Δ%| > 5%（维度 1 中级触发）→ 立即升级 Active
  • 任一账户 margin utilization > 80% → 进入 Active 评估

1.6 冷却规则

  • Standby → Active：立即触发，无 cool-down（避免延迟）
  • Active → Standby：所有触发解除 + 持续 1h 后才回 Standby
  • Active mode 切换（如 Standard → Defensive）：升级立即；降级有 30min 冷却

2.2 减仓规则（Orange / Red 触发时）

  • 按 单 leg Γ 排序（短到期天然 Γ 高，自动排在前面），取最大者
  • 大单 > 200 BTC notional 走 Deribit RFQ / Block，避免 order book 冲击
  • 单次减仓 ≤ 该 leg 持仓的 40%（防冲击 IV）
  • 减仓后立即重算 Γ_p；未达 < 0.25 继续循环
  • Red 状态执行加速版：拆单更小、PM / 风控 电话 + IM 双通道通知
安全区目标统一为 Γ_p < 0.25（Green 区），而不是回到触发阈值 0.30 边缘 —— 留出 buffer 防止立即再触发。

4优先级与互锁

Γ 优先于 Δ：触发对冲前先检查 Γ_p；若已 🟠 Orange 或 🔴 Red，先减 Γ 再做 Δ hedge。高 Γ 时频繁 Δ hedge 会"追着市场跑"，亏损放大。
